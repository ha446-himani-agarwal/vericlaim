Medical Fraud Detection System

VeriClaim

An fraud detection system that detects fraudulent medical insurance claims by analyzing CPT/HCPCS procedure codes, ICD-10 diagnosis codes, provider legitimacy, and billing address validity.

Detection Architecture
📋
Layer 1 - CMS Rule Engine
Deterministic · Instant · No API

Hardcoded CMS billing rules that catch clear-cut violations instantly. Checks procedure-diagnosis linkage, bundling requirements, and price benchmarks.

Diagnosis conditionals Bundling violations Implausible code combos Price benchmarks
🤖
Layer 2 - Claude AI
Contextual · claude-sonnet-4 · Anthropic

Contextual reasoning for contextual fraud patterns the rules cannot assess. Receives rule engine output as context and validates or escalates findings.

Clinical plausibility Upcoding detection Plain English explanation
🏥
Layer 3 - NPI Registry
Live · CMS NPPES · Public API

Real-time verification of the billing provider against the CMS National Plan and Provider Enumeration System. Confirms registration, active status, and specialty.

Provider registration Active status Specialty verification
📍
Layer 4 - Address Verification
Geocoding · Census Bureau · Claude Analysis

Geocodes the billing address via the US Census Bureau and classifies the location type. Residential or commercial billing addresses are flagged as fraud indicators.

Clinical / Residential / Commercial GPS coordinates
Fraud Typology
Upcoding
Billing a higher-cost procedure code than the service actually performed.
Code Padding
Appending unrelated procedure codes to inflate total reimbursement.
Phantom Billing
Billing for services or procedures that were never performed.
Diagnosis Mismatch
Procedure code billed without the required ICD-10 diagnosis code linkage.
Modifier Abuse (-59)
Misusing modifier -59 to bypass CMS bundling rules and bill separately.
Provider Fraud
Claims submitted under unregistered, inactive, or fictitious provider NPIs.
